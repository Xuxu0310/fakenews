3.3 Data Cleaning
-------------------------------------------------------------------------------

3.3.1 Text Normalization

Text normalization is the first step in preprocessing pipeline and involves several critical transformations to clean raw article content. The first step converts all text to lowercase to ensure consistency and reduce vocabulary size by treating identical words with different capitalizations as the same token. URLs and web addresses are systematically removed using regular expressions, as these elements typically do not contribute to semantic meaning relevant for fake news detection and may introduce noise into the vocabulary. Special characters are filtered to retain only alphanumeric characters and apostrophes, with multiple consecutive spaces replaced by single spaces to normalize whitespace. These cleaning steps reduce the vocabulary size while preserving the meaningful linguistic content that indicates credibility. The text cleaning function is implemented as follows:

```python
def clean_text(text):
    text = "" if pd.isna(text) else str(text)
    text = text.lower()                              # Convert to lowercase
    text = re.sub(r"http\S+|www\.\S+", " ", text)   # Remove URLs
    text = re.sub(r"[^a-z0-9\s']", " ", text)   # Remove special chars
    text = re.sub(r"\s+", " ", text).strip()    # Normalize whitespace
    return text
```


3.3.2 Tokenization
-------------------------------------------------------------------------------

Tokenization converts the cleaned text into individual word units using a regular expression pattern matching. Each article is broken down into a sequence of tokens that serve as the basic units for vocabulary construction and subsequent numerical encoding. The core tokenization function is implemented as follows:

```python
def tokenize(text):
    return re.findall(r"[a-z0-9']+", text.lower())
```

This approach identifies all alphanumeric sequences followed by apostrophes as individual tokens, allowing the system to handle contractions and possessive forms appropriately. The lowercase conversion ensures that the same word with different capitalizations is treated as a single token, reducing vocabulary redundancy.


3.3.3 Vocabulary Construction
-------------------------------------------------------------------------------

The vocabulary is built from the training set with a maximum size of 30,000 tokens and a minimum frequency threshold of 2 occurrences. This filtering ensures that common words are included while rare or noise tokens that appear fewer than twice are excluded from the vocabulary. The vocabulary building process is implemented as follows:

```python
def build_vocab(texts, max_vocab_size=30000, min_freq=2):
    counter = Counter()
    for text in texts:
        counter.update(tokenize(text))

    vocab = {"<PAD>": 0, "<UNK>": 1}
    for token, freq in counter.most_common():
        if freq < min_freq:
            continue
        if token in vocab:
            continue
        vocab[token] = len(vocab)
        if len(vocab) >= max_vocab_size:
            break
    return vocab
```

Special tokens are added to the vocabulary: `<PAD>` with ID 0 for padding sequences to a uniform length, and `<UNK>` with ID 1 for unknown words not present in the vocabulary during inference. This design allows the system to handle out-of-vocabulary tokens gracefully while maintaining consistent input dimensions across all samples.

3.3.4 Sequence Encoding
-------------------------------------------------------------------------------

Sequence encoding transforms tokenized text into fixed-length numerical vectors for neural network input. Each article is converted to a sequence of token IDs using the vocabulary lookup, with a maximum sequence length of 220 tokens. Articles shorter than 220 tokens are padded with `<PAD>` tokens to reach the required length, while longer articles are truncated to 220 tokens, keeping only the first 220 tokens. The sequence encoding function is implemented as follows:

```python
def encode_text(text, vocab, max_len=220):
    tokens = tokenize(text)
    ids = [vocab.get(tok, vocab["<UNK>"]) for tok in tokens[:max_len]]
    length = len(ids)
    if length < max_len:
        ids += [vocab["<PAD>"]] * (max_len - length)
    return ids, min(length, max_len)
```

This standardization ensures consistent input dimensions for all samples, which is necessary for efficient batch processing in neural networks. The choice of 220 as the maximum length was informed by examining the distribution of article lengths to ensure that the majority of articles are fully represented while maintaining computational efficiency.

3.4 Dataset Split
-------------------------------------------------------------------------------

After preprocessing, the dataset is divided into three subsets following a stratified sampling strategy to maintain consistent class distributions across all splits. This approach ensures that each subset accurately reflects the overall class balance and prevents potential bias that could arise from random splitting. The training set provides the primary data for model learning, the validation set is used for hyperparameter tuning and early stopping decisions, and the test set serves as a held-out set for final unbiased performance evaluation.

The dataset distribution across the three subsets is summarized in the following table:

+----------------+-------+----------+----------+
| Subset         | Count | Percentage | Purpose  |
+----------------+-------+----------+----------+
| Training        | 3,324 | 70.0%     | Model learning and weight updates  |
| Validation      | 712   | 15.0%     | Hyperparameter tuning and early stopping  |
| Test           | 713   | 15.0%     | Final unbiased evaluation  |
+----------------+-------+----------+----------+
| Total           | 4,749 | 100.0%    |  |
+----------------+-------+----------+----------+

The data splitting is implemented using scikit-learn's `train_test_split` function with stratified sampling:

```python
def split_data(df, test_size=0.15, val_size=0.15, seed=42):
    train_df, temp_df = train_test_split(
        df,
        test_size=(test_size + val_size),
        random_state=seed,
        stratify=df["label"]
    )
    relative_test = test_size / (test_size + val_size)
    val_df, test_df = train_test_split(
        temp_df,
        test_size=relative_test,
        random_state=seed,
        stratify=temp_df["label"]
    )
    return train_df, val_df, test_df
```

After initial loading, the dataset undergoes additional cleaning procedures to remove missing values and duplicate entries based on the combination of text and label pairs. This preprocessing ensures data quality before model training and prevents potential issues that could arise from redundant or corrupted data points.
